# My Preferences

## Development

- **Mobile Framework:** Prefer React Native over Flutter
- **Language:** TypeScript > JavaScript always
- **Style:** Functional programming over OOP
- **Testing:** TDD when possible, at least unit tests

### Why React Native?

- JavaScript/TypeScript ecosystem
- Hot reload during development
- Cross-platform with single codebase
- Large community and libraries

## Tools & Setup

### Editor
- **Primary:** VS Code
- **Extensions:** ESLint, Prettier, GitLens
- **Theme:** Dark+ (default dark)
- **Font:** Fira Code with ligatures

### Terminal
- **macOS:** iTerm2 with Oh My Zsh
- **Windows:** Windows Terminal
- **Shell:** Zsh with powerlevel10k

### Version Control
- **Git** with conventional commits
- **GitHub** for hosting
- **Linear** for issue tracking

### Notes
- **Obsidian** for knowledge management
- Daily notes for journaling
- Zettelkasten method
- Weekly reviews

## Learning Style

### How I Learn Best

1. **Learn by building projects**
   - Start with small proof-of-concept
   - Iterate and expand
   - Refactor as I understand better

2. **Reference documentation frequently**
   - Official docs over tutorials
   - Keep docs open while coding
   - Build mental models

3. **Take notes in markdown**
   - Write as I learn
   - Link related concepts
   - Review and refine later

4. **Review notes weekly**
   - Sunday evening review session
   - Update stale information
   - Archive completed projects

### Resources I Trust

- Official documentation (React Native, TypeScript)
- Kent C. Dodds blog (React patterns)
- Martin Fowler (architecture)
- Stack Overflow (specific problems)

## Work Preferences

### Optimal Environment

- **Time:** Early morning (6am-10am) for deep work
- **Music:** Instrumental (lo-fi, classical) or silence
- **Breaks:** Pomodoro technique (25min work, 5min break)
- **Location:** Home office, coffee shop for variety

### Communication

- **Async first:** Written communication preferred
- **Meetings:** Only when necessary, with clear agenda
- **Documentation:** Over-document rather than under-document
